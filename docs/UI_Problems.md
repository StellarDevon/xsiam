## 🔴 P1 — 风格极度不统一（4 套并存）

扫完所有页面，发现同时存在 **4 套不同视觉风格**：

| 风格 | 涉及页面 | 问题描述 |
|------|----------|----------|
| **A：深色 Header + 白色内容区** | 告警、事件、漏洞、动作中心、设备、检测规则、报表、剧本 | 白色大面积内容区，与深色侧边栏割裂 |
| **B：纯深色全屏** | ETL 流水线、查询中心 | 与侧边栏融合，沉浸感好，是目标风格 |
| **C：浅灰全屏** | 身份风险、Agentix、资产 | 浅色内容区，像另一个独立产品 |
| **D：白色大背景** | Agentix Command Center | 白底 + 彩色 Agent 圆圈，像 PPT 示意图 |

**根本原因**：
- PageHeader 组件强制使用白色背景条，造成视觉割裂
- 各页面 `--bg-primary` / `--bg-secondary` 使用不一致
- 部分页面是亮色主题，部分是暗色主题，混用

**修复方向**：
- 去掉独立的白色 PageHeader 条，页面标题改为侧边栏高亮 + 页面内 Tab 级别
- 内容区背景统一使用 `var(--bg-primary)`（当前已定义为深色）

## 🔴 P1 — 交互设计问题

### 1. 表格行高过大
- **涉及**：告警、事件、漏洞、动作中心
- **问题**：每行 60px+，一屏只显示 10 条左右，信息密度极低
- **对比**：Datadog / Splunk 通常 36–40px
- **修复**：表格行高统一改为 40px，padding 上下各 10px

### 2. Stats 卡片无设计感
- **涉及**：告警、事件、报表头部统计
- **问题**：纯数字 + 灰色方块，无颜色区分、无 sparkline、无趋势箭头
- **修复**：加颜色语义（红/黄/绿）、加小型折线趋势、数字加粗放大

### 3. Agentix 页面色彩混乱
- **问题**：白色背景 + 五颜六色 Agent 圆圈（橙、紫、金、蓝绿），与整体深色产品完全割裂
- **修复**：背景改为深色，Agent 圆圈颜色降饱和度，统一色板

### 4. 身份风险页面大量空白
- **问题**：5 条数据占全屏但下方 1/2 面积空白，没有补充信息
- **修复**：加右侧详情面板，或展示用户行为时间线

### 5. 页面 Header 占用冗余空间
- **问题**：每个页面顶部都有一条白色 header（如"告警管理 · 概览"），与内容重复，浪费约 48px 高度
- **修复**：去掉独立 header 条，页面标题融入 Tab 栏左侧

### 6. 中英文混用严重
- **涉及**：动作中心（Action Log、Live Terminal、Script Library、SOAR 仪表盘）、威胁情报（Format、Generated、Source/Feed、Related Incidents）、资产（Agent Status）
- **修复**：全部改为中文，或全部改为英文，保持一致

### 7. 按钮/操作样式混乱
- **问题**：同一页面内出现多种按钮样式，部分操作按钮和主操作按钮视觉权重相同
- **修复**：严格区分 Primary（蓝色实心）/ Secondary（镂空）/ Danger（红色/图标）三级

## 🟡 P2 — 功能重复，建议合并

| 重复项 | 当前状态 | 建议 |
|--------|----------|------|
| **威胁情报 + IOC 管理** | 两个独立页面，IOC 路由还挂了 | 合并为一个页面，IOC 作为威胁情报的子 Tab |
| **剧本 + 动作中心** | 剧本单独菜单，与动作中心功能高度关联 | 剧本并入动作中心，作为第三个 Tab |
| **Agentix + 案例** | 案例路由挂了；Agentix 有 Case Investigation Agent | 案例作为 Agentix 的子视图 |
| **检测规则 + ETL 规则** | 两套规则管理，设计语言不同 | 放入"规则引擎"父菜单下统一管理 |
| **设备 + 资产** | 资产是逻辑资产，设备是 Agent 管理，大量重叠 | 资产管理作主页面，设备作为其中一个 Tab |
